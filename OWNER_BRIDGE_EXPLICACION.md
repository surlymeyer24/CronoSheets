# 🏗️ Arquitectura - Owner Bridge y Solución de Permisos\n\n## El Problema Original\n\n```\nUsuario (trabajador)\n  ↓\n  Abre Sidebar en el Spreadsheet\n  ↓\n  Intenta desproteger una hoja\n  ↓\n  ❌ ERROR: \"No tienes permiso para modificar esta protección\"\n  ↓\n  ¿Por qué? Porque:\n  - Las protecciones fueron creadas por el PROPIETARIO\n  - Solo el propietario (o quiénes el propietario agregó) pueden quitarlas\n  - El trabajador no fue agregado como editor de la protección\n```\n\n---\n\n## La Solución: Owner Bridge Pattern\n\nEn lugar de que el usuario intente desproteger algo que no puede, **delega la tarea al propietario**:\n\n```\nCapas de Solución:\n\nCapa 1: Usuario ejecuta función\n  ├─ Usuario invoca: agregarGuardiasBatch()\n  │  └─ Esta función típicamente fallaría por permisos\n  │\n  └─ ✅ SOLUCIÓN: Primero intenta vía Owner Bridge\n\nCapa 2: Backend intenta vía Owner Bridge (la mayoría de veces funciona)\n  ├─ backend.js llama a: desprotegerHoja()\n  │  └─ Que llama a: llamarOwnerBridge_('desprotegerHoja', ...)\n  │     └─ ✅ Esto FUNCIONA porque el owner tiene permisos\n  │\n  └─ Si el Owner Bridge falla por alguna razón:\n     └─ Capa 3: Intenta localmente (fallback)\n\nCapa 3: Fallback local (para cuando Owner Bridge no está disponible)\n  └─ El usuario intenta desproteger como antes\n  └─ Ahora con MEJOR logging para entender qué falló\n```\n\n---\n\n## Cómo Funciona el Owner Bridge\n\n### Arquitectura de 2 Proyectos\n\n```\n┌─────────────────────────────────────────────────────────┐\n│  PROYECTO MAIN (backend.js)                             │\n│  - Donde el usuario ejecuta cosas                        │\n│  - NO tiene que ser propietario del spreadsheet         │\n│  - Llama al Owner Bridge cuando necesita permisos       │\n│                                                         │\n│  Función: desprotegerHoja()                             │\n│    1. Intenta vía Owner Bridge                          │\n│    2. Si falla, intenta localmente                      │\n└─────────────────────────────────────────────────────────┘\n                      ↓ (HTTP POST)\n        [LLAMADA REMOTA CON TOKEN]\n                      ↓\n┌─────────────────────────────────────────────────────────┐\n│  PROYECTO OWNER (owner/Código.js)                       │\n│  - Desplegado como Web App                              │\n│  - DEBE ser del propietario del spreadsheet             │\n│  - Solo acepta llamadas con TOKEN correcto              │\n│                                                         │\n│  Función: desprotegerHojaOwner()                        │\n│    - Ejecuta como el propietario                        │\n│    - Puede desproteger cualquier protección            │\n│    - Retorna resultado al main                          │\n└─────────────────────────────────────────────────────────┘\n```\n\n### Flujo de una Llamada al Owner Bridge\n\n```javascript\n// EN MAIN:\nconst result = llamarOwnerBridge_('desprotegerHoja', {\n  spreadsheetId: ss.getId(),\n  sheetName: 'Resumen'\n});\n\n// Esto hace:\n// 1. Toma la URL del owner desde propiedades del script\n// 2. Toma el TOKEN desde propiedades del script\n// 3. Arma un JSON:\n//    {\n//      accion: 'desprotegerHoja',\n//      spreadsheetId: '1234...',\n//      sheetName: 'Resumen',\n//      token: 'abc123...'\n//    }\n// 4. Hace POST HTTPS a la URL del owner\n// 5. El owner recibe, valida el token\n// 6. Si es válido, ejecuta desprotegerHojaOwner()\n// 7. Retorna el resultado al main\n```\n\n---\n\n## Secuencia Detallada del Flujo Arreglado\n\n### ANTES (Fallaba):\n```\n1. Usuario sin permisos intenta desproteger\n   ↓\n2. desprotegerHoja() intenta vía Owner Bridge\n   ├─ Funciona? → ✅ OK, termina\n   └─ No? → Intenta localmente\n      ↓\n3. Intento local falla silenciosamente\n   ↓\n4. Usuario no sabe por qué falló ❌\n```\n\n### DESPUÉS (Funciona):\n```\n1. Usuario intenta desproteger\n   ↓\n2. desprotegerHoja() intenta vía Owner Bridge\n   ├─ Funciona? → ✅ OK, log: \"Owner Bridge desprotegió\"\n   │             termina aquí, sin intentar local\n   │\n   └─ No funciona? → Log detallado del error\n      ↓\n3. Intenta localmente CON LOGS\n   ├─ Funciona? → ✅ OK, log: \"Desprotección local exitosa\"\n   │\n   └─ No? → Log detallado:\n      └─ \"Error: ${e.message}\"\n      └─ \"No se tienen permisos para desproteger\"\n      ↓\n4. Usuario SABE exactamente qué falló y por qué 📝\n```\n\n---\n\n## Cómo Validar que Owner Bridge Está Funcionando\n\n### Test 1: Configuración\n```javascript\n// En la consola del PROYECTO MAIN:\nverOwnerBridgeConfig()\n\n// Debería retornar:\n// {\n//   habilitado: true,\n//   url: \"https://script.google.com/macros/s/AKfy...\",\n//   tokenConfigurado: true,\n//   tokenPreview: \"abc123de...\"\n// }\n```\n\n### Test 2: Conectividad\n```javascript\n// Aún en PROYECTO MAIN:\ntestOwnerBridgeConexion()\n\n// Debería retornar:\n// {\n//   success: true,\n//   message: \"Conexión OK con Owner Bridge\",\n//   detail: {...}\n// }\n```\n\n### Test 3: Acceso Real\n```javascript\n// Prueba real:\n// Abre el Sidebar con un usuario que NO es propietario\n// Intenta agregar una guardia\n// Revisa los logs en Extensiones → Apps Script → Ejecuciones\n// Busca: \"Owner Bridge desprotegió exitosamente\" ✅\n```\n\n---\n\n## Cambios Específicos que Permitieron Esto\n\n### 1. **Desprotección más robusta**\n\n**Antes:**\n```javascript\nconst viaOwner = llamarOwnerBridge_('desprotegerHoja', {...});\nif (viaOwner.status === 'ok') {\n  SpreadsheetApp.flush();\n  return; // Sin log útil\n}\n```\n\n**Después:**\n```javascript\nif (viaOwner.status === 'ok') {\n  Logger.log('  ✓ Owner Bridge desprotegió exitosamente');\n  SpreadsheetApp.flush();\n  return; // Claro en los logs\n}\n\nif (viaOwner.status !== 'disabled') {\n  Logger.log('  ✗ Owner Bridge falló: ' + JSON.stringify(viaOwner));\n  // Muestra exactamente qué salió mal\n}\n```\n\n### 2. **Protección con más editores**\n\n**Antes:**\n```javascript\np.addEditor(email);        // usuario actual\np.addEditor(adminEmail);   // 1 admin\n// Solo 2 editores\n```\n\n**Después:**\n```javascript\np.addEditor(email);           // usuario actual\np.addEditor(adminEmail);      // 1 admin\np.addEditor(emailDes);        // 2 admin adicional\n// 3 editores = más flexible\n```\n\n### 3. **Reintentos en reprotección**\n\n**Antes:**\n```javascript\ntry {\n  p.remove();\n  removidas++;\n} catch (e) {\n  // Silencio, no sabemos qué pasó\n}\n```\n\n**Después:**\n```javascript\ntry {\n  p.remove();\n  removidas++;\n  Logger.log('      ✓ Removida');\n} catch (e) {\n  errores++;\n  Logger.log('      ✗ Error: ' + e.message);\n  // Sabes exactamente dónde falló\n}\n```\n\n---\n\n## Diagrama de Decisión\n\n```\n¿Puedo desproteger?\n    ↓\n    ├─→ SÍ (soy el propietario)\n    │   └─ Haz lo local\n    │\n    └─→ NO (soy usuario normal)\n        ├─ ¿Owner Bridge disponible?\n        │  ├─→ SÍ → Usa Owner Bridge ✅ (SOLUCIÓN ACTUAL)\n        │  └─→ NO → Intenta local, espera a que propietario\n        │\n        └─ ¿Soy agregado como editor de la protección?\n           ├─→ SÍ (nuevo después del fix) → Puedo hacerlo\n           └─→ NO → Espera al propietario o usa Owner Bridge\n```\n\n---\n\n## Resumen: Qué Hace Owner Bridge\n\n| Aspecto | Descripción |\n|---------|-------------|\n| **Quién** | Usuario → Backend → Owner Cloud Function |\n| **Qué** | Ejecuta operaciones de protección como el propietario |\n| **Cuándo** | Cuando el usuario no tiene permisos |\n| **Por qué** | El propietario SIEMPRE puede modificar sus propias protecciones |\n| **Cómo** | HTTP POST con token de seguridad |\n| **Resultado** | El usuario no necesita ser propietario para hacer cambios válidos |\n\n---\n\n**Conclusión:** El Owner Bridge convierte tu sistema de roles de \"solo el propietario puede hacer cambios\" a \"cualquiera puede hacer cambios válidos, el propietario solo valida\".\n
